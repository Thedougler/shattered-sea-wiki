# Adventure Structure Reference

WotC-style anatomy for D&D adventures. Use this when building chapter-structured campaigns or adapting published-adventure formatting. For wiki-voice entity pages, use `wiki-dnd/PREP.md` and entity templates instead.

---

## Front Matter

- Title and credits
- Table of contents (chapter breakdown, appendices, maps)
- **Introduction**
  - Adventure background — the story's foundation, what forces are in motion
  - Target level range and expected session count
  - Prerequisites
  - Recommended character types

---

## Core Structure

### Adventure Synopsis

High-level overview of expected flow — major story beats, key NPCs, the hook that pulls characters in. Show the connective tissue between scenes even if the campaign is non-linear.

### Adventure Hooks

- Primary hook tied to player character motivations
- 2–3 alternate entry points
- Connection to broader campaign (if applicable)

### Chapter Breakdown

Each chapter includes:

#### Location / Scene Description

- **Read-aloud text** — detailed, multi-sensory, present tense. No hidden info. Also serves as AI image prompt.
- **DM notes** — mechanical info, secrets, conditional outcomes, tactical guidance
- **Image placeholder** — link for future artwork

#### Encounters

Balance encounter types across chapters:

- Combat (with stat blocks or references)
- Social / roleplay
- Skill challenges
- Puzzles / traps

#### NPCs

- Unique description if homebrew; stat block reference if canon
- Personality traits, motivations, goals
- Roleplay guidance
- Image prompt

#### Treasure & Rewards

- Loot tables, magic items
- XP or milestone guidance

---

## Climax & Conclusion

### Final Encounter

- Most decisive encounter; dramatically meaningful stakes
- Players should care about failure, not just success
- Multiple resolution paths
- Consequences of both success and failure clear

### Resolution & Epilogue

- Wrap-up scenarios
- Campaign hooks for future play
- Character rewards and advancement

---

## Back Matter (Appendices)

- New monsters (full stat blocks)
- New magic items
- Handouts
- Random encounter tables
- NPC roster with page references

---

## Formatting Conventions

**Read-aloud text:**

```
> Present tense. Engage sight, sound, smell, touch. No mechanical info.
> Detailed enough to inspire imagination; concise enough to read aloud.
```

**DM notes:** Regular text. Contains mechanics, secrets, conditional outcomes, cross-references.

**Stat blocks:** Standardized format with DnDBeyond page references for published creatures.

**Maps:** Mermaid `flowchart LR` diagrams for spatial relationships; image placeholder link for artwork.

**Skill check callouts:** `[!skill-check]` callout type. Include DC, attribute, and what success/failure reveals.

**Secret callouts:** `[!secret]` callout type for GM-only information embedded inline.
