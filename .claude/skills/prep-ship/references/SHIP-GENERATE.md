# Ship Bastion — Content Generation

Load this sub-document when generating new ship-related content: facilities, magical enhancements, sea bastion events, or edge case rulings. For rules reference during generation, also read `references/SHIP-RULES.md`.

---

## New Facilities

1. Find the closest 2024 DMG facility as a mechanical base
2. Give it a nautical name and one sentence of flavour
3. Assign space units: **1** for personal/compact, **2** for operational/large
4. Note if it requires a hireling with a specific crew role
5. Flag if it's a special facility (Tier 2+ only)

The full facility name map is in `references/SHIP-RULES.md → Facility List`. Don't invent a facility whose slot is already filled by an existing equivalent.

---

## New Magical Enhancements

Tag every item `[RAW]` (2024 item used directly or reskinned) or `[HB]` (homebrew). Items should do **one thing cleanly**.

**Price anchors:**

| Rarity | Price Range |
|---|---|
| Common | 150–400gp |
| Uncommon | 600–1,500gp |
| Rare | 3,000–12,000gp |
| Legendary | 20,000gp+ |

Match rarity to availability tier in the campaign setting. Common items are in Tidefall. Legendary items are late-campaign only. Catarina Da'Virelli is the primary access point for rare and unusual items.

## New Sea Bastion Events

Replacement events for the 2024 Bastion Event table when the ship is underway. Each replacement should:

- Be a sea-plausible equivalent to the location-specific 2024 event it replaces
- Be resolvable through combat, social, **or** investigation — not all three
- Tie to existing factions and locations where possible
- Not require the party to be in port

**Existing replacements (in `RULES.md → Bastion Events`):**

| 2024 Event Type | Shipboard Equivalent |
|---|---|
| Local threat / attack | Hostile vessel tracking the ship |
| Beneficial contact | Passing ship offers information, trade, or opportunity |
| Structural damage | Storm or incident damages a facility |
| Windfall | Favourable current shortens voyage by 1d4 days |
| Hireling issue | Hireling problem per 2024 hireling event rules |

When adding new events, use this table as the structural model. Keep the consequence proportional to the 2024 equivalent.

## Ship Variants Reference

When generating a named ship or NPC vessel, apply **one primary variant**. Combining variants is possible but each compounds the tradeoff — don't do it unless there's a strong narrative reason.

| Variant | Key Gain | Key Loss | Cost Modifier |
|---|---|---|---|
| Speed-Built | +15–20 mi/day good wind | −⅓ cargo, −¼ guns | +10–15% |
| Armed | +25–30% gun mounts | −10 mi/day, −⅕ cargo | +10% |
| Defended | +20% HP, AC +1 | −5 mi/day, −⅒ cargo | +15% |
| Cargo | +40–50% cargo | −½ guns, −10 mi/day | same or below |
| Crew-Optimized | +1 facility space unit, hireling loyalty advantage | — | +5–10% |

Refit cost to convert an existing ship: 10–20% of base cost, 2–8 weeks at a shipyard.

## Edge Case Rulings

When `references/SHIP-RULES.md` is silent, apply this precedence:

1. **2024 DMG bastion rule** — if it makes narrative sense at sea
2. **Spirit of the existing adaptations** — shared direction, no level gate, crew-quality-as-defense — when the DMG rule breaks at sea
3. **Player agency** — if ambiguous, lean toward the ruling that gives players a meaningful choice
