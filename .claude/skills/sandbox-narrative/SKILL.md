## Core Principle

Narrative devices are tools for world-building, not plot control. You place loaded guns —
players decide when they fire.

**Anti-railroading check:** Before applying any device, ask: *does this require a specific
player choice to pay off?* If yes, it's a railroad. Restructure so multiple player paths
lead to equally valid outcomes.

## Universal Toy Properties

Any prep element players can't help but interact with satisfies all four:

1. **Clarity** — instantly understandable, zero lore dump required
2. **Agency** — fight it, negotiate it, exploit it, ignore it (with consequences for each)
3. **Reactivity** — moves even when untouched; the world pushes back
4. **Portability** — works in any town, any party

**Three Hooks Rule:** Every major toy needs moral, personal, and opportunistic entry points. If only one type of player would care, the toy misses half the table.

---

## Prep Philosophy

1. Prep situations, not scenes. Define pressures + timelines, not outcomes.
2. Every element acts on its own logic when players aren't watching.
3. Player characters are the center of mass. Everything else orbits them.
4. If you removed the players and the world wouldn't change — you wrote a plot, not a sandbox.

**Don't wait for players to find content.** The cult kidnapped the paladin's mentor.
The gang hired the rogue's old crew. Make it personal before the session starts.

---

## PC Gravity

Every prep element must pull on at least one PC's internal tensions.

**For each PC, identify:**
- **Two Dials** — two core behavioral axes (e.g., family loyalty / reckless ambition). Internal tensions, not surface traits.
- **Terminal Node** — deepest long-term desire. Asymptotic — approaches but never arrives.
- **Active Friction** — what currently blocks them. This is where you place toys.

**The Gravity Filter:** Does this element pull on at least one PC's dials or terminal node?
- Yes → include it, note which PC and how
- No → cut it, or retrofit a connection

## Narrative Device Quick Reference

Read `references/narrative-devices.md` for full treatment of each.

| Device | Sandbox Application |
|---|---|
| Chekhov's Gun | Seed elements with future weight; don't aim them. Players pull triggers. |
| Foreshadowing | Hint at conditions and threats, not events. Multiple possible futures. |
| Ticking Clock | Primary anti-passivity tool. Factions pursue goals on timelines. Inaction compounds. |
| Three Clue Rule | Any conclusion players must reach: three distinct, mechanically discoverable clues. |
| Iceberg | Build 3x more world than will ever surface. Depth makes the visible tip feel real. |
| In Medias Res | Start sessions mid-action, not at the beginning. Skip the boring part. |

## Three Clue Rule — Structural Requirement

Any situation where players must reach a hidden conclusion to engage: three distinct,
mechanically discoverable clues must exist before the situation file is complete.
Redundancy rule: clues must span different nodes. Two clues in the same room fails.

Before closing any situation with a hidden conclusion:
- [ ] One conclusion named
- [ ] Clue 1: location + discovery mechanic
- [ ] Clue 2: different location + different mechanic
- [ ] Clue 3: third location or NPC + third mechanic
- [ ] At least two clues reachable without combat
